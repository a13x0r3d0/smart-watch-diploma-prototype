# Watch Simulator

`watch_sim` is a local C++ smartwatch simulator that behaves like a simple ESP32-style device. It keeps watch state in memory, shows a terminal-based watch screen, serves a browser-based watch UI, and exposes an HTTP API that the Android emulator can call through the host machine.

## Features

- Terminal OLED-like screen with four watch screens: main, steps, battery, alarm
- Browser-based watch UI at `http://127.0.0.1:8080/`
- Simulated watch state:
  - current time
  - battery level
  - steps count
  - alarm time
  - authentication status
- Local HTTP API on `http://127.0.0.1:8080`
- Background simulation:
  - steps increase every few seconds
  - battery drains slowly over time
- Simple keyboard controls

## Data Source In The Prototype

The simulator does not use real sensors. Watch values are produced by an internal software simulation layer.

In this prototype:

- steps are generated by mock logic that increases the counter over time
- battery level is reduced gradually by a battery-drain simulation
- temperature is changed by a small bounded random variation

This keeps the demo stable and fully local while preserving the same API and state structure that a real smartwatch would use.

## Build And Run

Requirements:

- macOS or Linux
- `clang++` or another C++17 compiler
- terminal

Commands:

```bash
cd watch_sim
make
./watch_simulator
```

Or:

```bash
cd watch_sim
make run
```

Then open the watch UI in a browser:

```text
http://127.0.0.1:8080/
```

## Keyboard Controls

- `n` = next screen
- `p` = previous screen
- `l` = toggle lock/unlock
- `q` = quit

## API Endpoints

### `GET /status`

Example response:

```json
{
  "time": "20:30",
  "day": "Monday",
  "date": "21.04.2026",
  "screen": "Main",
  "battery": 87,
  "steps": 1534,
  "temperature": 24,
  "alarm": "07:30",
  "authenticated": true
}
```

### `POST /auth`

Request:

```json
{
  "pin": "1234"
}
```

Response:

```json
{
  "success": true,
  "authenticated": true
}
```

### `POST /sync-time`

Requires authentication.

Request:

```json
{
  "time": "20:31"
}
```

### `POST /alarm`

Requires authentication.

Request:

```json
{
  "time": "07:30"
}
```

### `POST /lock`

Optional helper endpoint for the app lock button.

Request:

```json
{
  "locked": "true"
}
```

### `POST /screen`

Switches the visible watch screen in the terminal and browser UI.

Request:

```json
{
  "action": "next"
}
```

## Manual API Testing

Authenticate:

```bash
curl -X POST http://127.0.0.1:8080/auth \
  -H "Content-Type: application/json" \
  -d '{"pin":"1234"}'
```

Get status:

```bash
curl http://127.0.0.1:8080/status
```

Set alarm:

```bash
curl -X POST http://127.0.0.1:8080/alarm \
  -H "Content-Type: application/json" \
  -d '{"time":"07:30"}'
```
