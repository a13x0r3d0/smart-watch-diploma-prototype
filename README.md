# Smart Watch Diploma Prototype

This repository contains a runnable diploma-project prototype of an intelligent smart watch system with authentication through a mobile app.

The prototype is intentionally local-first and demo-friendly:

- no physical hardware required
- no BLE required
- no cloud backend
- communication happens over local HTTP

## Architecture

The monorepo contains two subprojects:

### `watch_sim/`

A local C++ smartwatch simulator that acts like a simplified ESP32-style device.

- stores internal watch state in memory
- simulates battery drain and step updates
- displays watch screens in the terminal
- exposes HTTP endpoints on the host machine
- serves a local browser UI that replaces the physical watch display for demos

### `watch_firmware_arduino/`

An ESP32/Arduino IDE firmware version of the smartwatch logic.

- structured as an Arduino sketch with `setup()` and `loop()`
- ready to open in Arduino IDE
- uses ESP32 Wi-Fi and HTTP server libraries
- serves the same kind of watch API and watch UI from firmware
- demonstrates hardware readiness for a future physical smartwatch build

### `android_app/`

A Kotlin Android app that runs in the Android Studio emulator.

- authenticates with a PIN
- fetches watch state
- syncs watch time
- sets an alarm
- shows authentication state

### Communication

The emulator connects to the host machine through:

```text
http://10.0.2.2:8080
```

That points to the watch simulator running on the same computer.

## Why Arduino IDE + Web UI

This prototype intentionally combines two presentation layers for one smartwatch concept:

- `watch_firmware_arduino/` shows that the project is already structured like real ESP32 firmware with `setup()`, `loop()`, Wi-Fi setup, HTTP routes, and screen-rendering logic
- `watch_sim/` provides a stable local web-based watch display that replaces the physical OLED during the diploma defense

This combination is useful for a diploma project because it proves two things at once:

- the software is ready to be migrated onto real watch hardware
- the full system can still be demonstrated locally, reliably, and repeatedly without buying or wiring a physical device

In presentation terms:

- Arduino IDE demonstrates hardware readiness
- Web UI demonstrates the watch display and live interaction
- Android app demonstrates the mobile client and authentication flow

## Where The Watch Data Comes From

In the current diploma prototype, values such as:

- steps
- temperature
- battery level

are generated by a local simulation layer inside the watch backend.

This means the project does not yet depend on real physical sensors. Instead:

- steps are increased by a mock step sensor
- temperature changes slightly over time using a mock temperature sensor
- battery level slowly decreases using a mock battery model

This is intentional for demo stability and local execution without hardware.

For the defense, this can be explained as:

- the current prototype uses software-based sensor simulation
- the data model and API are already designed as they would be in a real smartwatch
- in a hardware version, the mock sensor functions can be replaced with real readings from:
  - an accelerometer for step detection
  - a temperature sensor
  - a battery ADC, PMIC, or fuel-gauge chip

## API Overview

### `GET /status`

Response:

```json
{
  "time": "20:30",
  "day": "Monday",
  "date": "21.04.2026",
  "battery": 87,
  "steps": 1534,
  "temperature": 24,
  "alarm": "07:30",
  "authenticated": true
}
```

### `POST /auth`

Request:

```json
{
  "pin": "1234"
}
```

Response:

```json
{
  "success": true,
  "authenticated": true
}
```

### `POST /sync-time`

Request:

```json
{
  "time": "20:31"
}
```

### `POST /alarm`

Request:

```json
{
  "time": "07:30"
}
```

## How To Run

## 1. Start The Watch Simulator

Requirements:

- macOS or Linux
- `clang++` or another C++17 compiler

Commands:

```bash
cd watch_sim
make
./watch_simulator
```

The simulator starts a local HTTP server on port `8080`.

## 2. Run The Android App

Requirements:

- Android Studio
- Android SDK installed from Android Studio
- JDK configured in Android Studio

Steps:

1. Open `android_app` in Android Studio.
2. Wait for Gradle sync to finish.
3. Start an Android emulator.
4. Run the app.

## 3. Test The Full Flow

1. Start `watch_simulator`.
2. Launch the Android app in the emulator.
3. Enter PIN `1234`.
4. Open the dashboard.
5. Tap `Fetch Watch Status`.
6. Tap `Sync Time`.
7. Enter an alarm like `07:30`.
8. Tap `Set Alarm`.
9. Tap `Lock Watch` to demonstrate protected actions.

## Demo Scenario

Use this presentation flow:

1. Show the watch simulator terminal with changing steps and battery.
2. Show that the watch is initially locked.
3. Open the Android app and enter PIN `1234`.
4. Fetch live watch status in the dashboard.
5. Sync time from the phone to the watch.
6. Set a new alarm.
7. Lock the watch again and show that authentication is required.
